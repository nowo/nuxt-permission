## Types

Import from `'nuxt-permission'`. Most are extended via declaration merging — see [Type extension](/en/guide/type-extension).

| Type | Description |
| :-- | :-- |
| `PermissionMap` | Empty interface; **extend it** to declare your permission keys |
| `PermissionKey` | The union of `PermissionMap`'s values if extended, otherwise `string` |
| `PermissionButton` | Shape of a folded button node (value of `meta._permission`); extend to type its fields |
| `PermissionMenu` | A menu node produced by `normalizePermissionMenus` (route-level fields + `children` + `meta`) |
| `PermissionSource` | The data source signature: `(ctx) => PermissionMenu[]` (sync or async) |
| `PermissionSourceContext` | The `ctx` injected into the source: `{ setPermissionList, setMenuList }` |

## Module options

Configured under the `permission` key in `nuxt.config.ts` — see [Configuration](/en/getting-started/configuration) for the full table.

| Option | Type | Default |
| :-- | :-- | :-- |
| `enabled` | `boolean` | `true` |
| `static` | `string \| string[]` | `['/', '/login']` |
| `source` | `string` | `'permission'` |
| `routeFields` | `string[]` | `[]` |
| `group` | `'redirect' \| 'navigate'` | `'redirect'` |
