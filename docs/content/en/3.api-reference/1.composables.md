Everything under `runtime/composables` is **auto-imported** â€” usable in templates and scripts without importing.

## `definePermissionSource`

```ts
function definePermissionSource(source: PermissionSource): PermissionSource
```

Type helper for the data source (the default export of `<srcDir>/permission.ts`). Returns the source unchanged; it exists only to type the `ctx` argument. See [Data source](/en/guide/data-source).

## `hasPermission`

```ts
function hasPermission(perm: PermissionKey | PermissionKey[], isAll?: boolean): boolean
```

Checks the current user's permissions. Reactive.

- `perm` â€” a single key or an array of keys.
- `isAll` â€” `false` (default): any key matches. `true`: every key must match.
- An **empty array** always returns `false`.

See [Permission control](/en/guide/permission-control).

## `normalizeMenus`

```ts
function normalizeMenus<T = any>(
  list: T[],
  cb?: (item: T) => any | undefined | null | false
): PermissionMenu[]
```

Transforms the raw backend menu tree into route shape. Mark buttons with `_btn: true` in `cb` to fold them into the parent's `meta._permission`; return a falsy value to drop a node and its subtree. `cb` is optional. See [normalizeMenus](/en/guide/normalize-menus).

## `usePermissionState`

```ts
function usePermissionState(): {
  permissions: Ref<PermissionKey[]>
  menus: Ref<PermissionMenu[]>
  routesVersion: Ref<number>
  hasPermission: (perm: PermissionKey | PermissionKey[], isAll?: boolean) => boolean
  setPermissionList: (list: string[]) => void
  setMenusList: (tree: PermissionMenu[]) => void
  load: () => Promise<void>
  clear: () => void
}
```

The permission / menu state store and its operations.

| Member | Type | Description |
| :-- | :-- | :-- |
| `permissions` | `Ref<PermissionKey[]>` | Current permission list (reactive; narrows once you extend `PermissionMap`) |
| `menus` | `Ref<PermissionMenu[]>` | Current menu tree (for rendering the sidebar) |
| `routesVersion` | `Ref<number>` | Bumped after every route registration; depend on it in a `computed` to re-read `useRouter().getRoutes()` (which is not reactive) |
| `hasPermission` | `(perm, isAll?) => boolean` | Same implementation as the standalone function |
| `setPermissionList` | `(list: string[]) => void` | Write the permission state (call inside the data source) |
| `setMenusList` | `(tree: PermissionMenu[]) => void` | Write the menu state |
| `load` | `() => Promise<void>` | Re-run the source, write state, register routes (after login / manual refresh). **Client-only** |
| `clear` | `() => void` | Clear state and unregister the dynamic routes added by `load()` (for logout) |

### Reading the latest full route list

`useRouter().getRoutes()` is a non-reactive snapshot. To re-read it after dynamic registration, depend on `routesVersion`:

```ts
const { routesVersion } = usePermissionState()
const routeList = computed(() => {
    void routesVersion.value      // dependency: bumped after addRoute â†’ recompute
    return useRouter().getRoutes() // now up to date
})
```
