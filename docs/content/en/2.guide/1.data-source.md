Create `<srcDir>/permission.ts` (the file name is the `source` option, default `permission`). It default-exports a source that reads the auth state, fetches permissions / menus, writes state, and returns the menu tree. **The library calls it on first load / refresh, and you call `load()` after login — the same source runs both times.**

```ts [app/permission.ts]
export default definePermissionSource(async ({ setPermissionList, setMenuList }) => {
    const token = /* your own auth state */ useCookie('token').value
    if (!token) return []

    const { permissions, menus } = await $fetch('/api/user', {
        headers: { authorization: `Bearer ${token}` },
    })

    setPermissionList(permissions)

    // Mark buttons with the reserved `_btn` flag so they fold into the parent's meta._permission
    const tree = normalizePermissionMenus(menus, v => ({ ...v, _btn: `${v.type}` === '2' }))

    setMenuList(tree)
    return tree
})
```

`definePermissionSource`, `normalizePermissionMenus`, and the composables are all auto-imported.

## Register after login

Token storage is up to you; after a successful login call `load()`, then navigate:

```ts
const { load } = usePermissionState()

async function onLogin() {
    await $fetch('/api/login', { method: 'POST' }) // You: obtain and store the token
    await load() // Library: run the source, write state, register routes
    await navigateTo('/dashboard')
}
```

::callout{icon="i-lucide-info"}
`load()` is client-only. The SSR first-paint path registers via `router.options`, so you never call `load()` during SSR.
::

## Logout

```ts
const { clear } = usePermissionState()

function onLogout() {
    clear() // clears permission / menu state and unregisters the dynamic routes
}
```
