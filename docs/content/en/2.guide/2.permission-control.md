`hasPermission` is auto-imported and usable directly in templates. There is **no** `v-permission` directive — native `v-if` is reactive, composes with `v-else`, and needs no extra mechanism.

```vue
<template>
    <button v-if="hasPermission('menu-add')">
        Add
    </button>
    <button v-if="hasPermission(['menu-edit', 'menu-view'])">
        Edit
    </button>        <!-- any match -->
    <button v-if="hasPermission(['menu-edit', 'menu-view'], true)">
        Edit
    </button>  <!-- all match -->
</template>
```

## Signature

```ts
function hasPermission(perm: PermissionKey | PermissionKey[], isAll?: boolean): boolean
```

- A single key or an array of keys.
- `isAll` (default `false`): any match is enough. `true`: every key must match.
- An **empty array** always returns `false` — an empty requirement grants no access.

## Reading state

```ts
const { permissions } = usePermissionState()
// permissions is Ref<PermissionKey[]> — narrows to your union once you extend PermissionMap
```

::callout{icon="i-lucide-shield-alert" color="warning"}
`hasPermission` is a UX layer only. Enforce real authorization on the backend.
::
