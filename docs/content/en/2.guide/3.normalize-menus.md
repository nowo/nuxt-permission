`normalizeMenus` turns the raw backend menu tree into route shape:

- a node marked `_btn: true` in `cb` is **folded** into the parent's `meta._permission` (key = the `permission` field, value = the whole button node with the `_btn` marker stripped)
- a node with menu children is treated as a **group** and `redirect`s to its first child
- `cb` returning a falsy value **drops** the node and its subtree
- a backend `meta` object is **flattened** into `meta`

```ts
const tree = normalizeMenus(menus, v => ({ ...v, _btn: `${v.type}` === '2' }))
```

The second argument `cb` is optional; omit it when the backend data is already in shape and there are no button permissions to fold.

## Reserved keys

All library keys are `_`-prefixed and isolated from your backend data:

| Key | Where | Meaning |
| :-- | :-- | :-- |
| `_btn` | input (you set it in `cb`) | marks a node as a button to fold |
| `meta._permission` | output | folded button permissions on the parent |
| `meta._external` | output | flags an external link (rendered as `<a>`, not registered) |

## Menu path handling

- **External links** (`http(s)://`, protocol-relative `//`) are kept in the tree but not registered — render them as `<a>` (they carry `meta._external`).
- **Param pages**: a backend path may use bracket or colon syntax — `/detail/[id]` and `/detail/:id` both match `pages/detail/[id].vue`.
- **Query strings**: `/report?range=7d` registers the pathname `/report` while keeping the full value for the sidebar link.
- **Deduplication**: the same page reached from several menu entries registers once (first wins); every entry still shows in the menu tree.

## Migrating from 1.x

::callout{icon="i-lucide-triangle-alert" color="warning"}
**Breaking change in 1.2.** Buttons are now marked with `_btn` in `cb` instead of being inferred from `type === 'button'`, and folded permissions moved from `meta.permission` to `meta._permission`. Update your `cb` to set `_btn`, and read `route.meta._permission` instead of `route.meta.permission`.
::
