## Label layer — always compatible

The library never touches `name` / `meta`, so translating menu labels is entirely up to you. Have the backend return an i18n key (e.g. `menu.dashboard`) and render the sidebar with `{{ $t(item.meta?.title ?? item.name) }}`, or return localized text directly.

## Routing layer

Dynamic routes are registered at runtime with their **raw** path (`/dashboard`), so i18n's build-time prefix strategies do not generate localized variants for them.

| Strategy | Compatibility |
| :-- | :-- |
| `no_prefix` | ✅ Single route set, locale via cookie — **recommended** |
| `prefix_except_default` | ⚠️ Default locale works; non-default `/en/...` variants are not generated |
| `prefix` / `prefix_and_default` | ❌ All paths are prefixed, the raw path does not match |

::callout{icon="i-lucide-lightbulb"}
Use `strategy: 'no_prefix'` with `@nuxtjs/i18n`. Prefix strategies together with runtime dynamic routes are a known limitation.
::
