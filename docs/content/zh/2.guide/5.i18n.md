## 文案层 —— 始终兼容

库从不触碰 `name` / `meta`，因此菜单文案的翻译完全由你掌控。可以让后端返回 i18n key（如 `menu.dashboard`），侧边栏用 `{{ $t(item.meta?.title ?? item.name) }}` 渲染；或者后端直接返回本地化文本。

## 路由层

动态路由是在运行时以其**原始** path（`/dashboard`）注册的，所以 i18n 构建期的前缀策略不会为它们生成本地化变体。

| 策略 | 兼容性 |
| :-- | :-- |
| `no_prefix` | ✅ 单套路由、locale 走 cookie —— **推荐** |
| `prefix_except_default` | ⚠️ 默认 locale 可用；非默认的 `/en/...` 变体不会生成 |
| `prefix` / `prefix_and_default` | ❌ 所有 path 都带前缀，原始 path 匹配不上 |

::callout{icon="i-lucide-lightbulb"}
搭配 `@nuxtjs/i18n` 时请用 `strategy: 'no_prefix'`。前缀策略与运行时动态路由并存是已知限制。
::
