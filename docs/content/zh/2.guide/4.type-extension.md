meta 复用 vue-router 的 `RouteMeta`；权限 key 用可合并的 `PermissionMap`。

```ts
declare module 'vue-router' {
    interface RouteMeta {
        title?: string
        icon?: string
        // ... 你的 meta 字段；在 menu.meta 和 route.meta 上都有类型
    }
}

declare module 'nuxt-permission' {
    interface PermissionMap {
        keys: 'menu-add' | 'menu-edit' | 'menu-view' // hasPermission 会获得 key 补全
    }
    interface PermissionButton { // meta._permission 里按钮节点的字段
        id: number
        name: string
        permission: string
    }
}
```

## key 如何收窄

在你扩展 `PermissionMap` 之前，`PermissionKey` 回退为 `string`：

```ts
type PermissionKey = keyof PermissionMap extends never
    ? string
    : PermissionMap[keyof PermissionMap]
```

扩展 `PermissionMap` 后，`hasPermission('...')` 与 `route.meta._permission['...']` 的 key 都会**自动收窄**为你声明的联合，同时 `usePermissionState()` 返回的 `permissions` 变成 `Ref<PermissionKey[]>`。

::callout{icon="i-lucide-info"}
无需（也无法）重声明 `RouteMeta._permission` 去覆盖它——interface 声明合并不允许用不同类型重定义同名属性。改为扩展 `PermissionMap`，key 会自行收窄。
::
