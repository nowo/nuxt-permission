`normalizePermissionMenus` 把后端原始菜单树转换成路由形状：

- 在 `cb` 里标注 `_btn: true` 的节点会**折叠**进父级的 `meta._permission`（key = `permission` 字段，value = 整个按钮节点，`_btn` 标记会被剥离）
- 有 menu 子节点的节点视为**分组**，`redirect` 到它的第一个子节点
- `cb` 返回 falsy 值会**丢弃**该节点及其子树
- 后端的 `meta` 对象会被**摊平**进 `meta`

```ts
const tree = normalizePermissionMenus(menus, v => ({ ...v, _btn: `${v.type}` === '2' }))
```

第二个参数 `cb` 是可选的；当后端数据已是正确形状、且没有按钮权限需要折叠时可省略。

## 保留字段

库的所有字段都以 `_` 前缀，与你的后端数据隔离：

| 字段 | 位置 | 含义 |
| :-- | :-- | :-- |
| `_btn` | 输入（你在 `cb` 里设置） | 标记一个节点为待折叠的按钮 |
| `meta._permission` | 输出 | 折叠到父级上的按钮权限 |
| `meta._external` | 输出 | 标记为外链（渲染成 `<a>`，不注册路由） |

## 菜单 path 处理

- **外链**（`http(s)://`、协议相对的 `//`）保留在树里但不注册——请渲染成 `<a>`（它们带有 `meta._external`）。
- **参数页**：后端 path 支持中括号或冒号两种写法——`/detail/[id]` 与 `/detail/:id` 都能命中 `pages/detail/[id].vue`。
- **带 query**：`/report?range=7d` 注册 pathname `/report`，同时保留完整值供侧边栏链接使用。
- **去重**：同一页面被多个菜单项指向时只注册一次（首个生效）；菜单树里仍保留每个入口。

## 从 1.x 迁移

::callout{icon="i-lucide-triangle-alert" color="warning"}
**1.2 破坏性改动。** 按钮现在改为在 `cb` 里用 `_btn` 标注，不再从 `type === 'button'` 推断；折叠的权限也从 `meta.permission` 移到了 `meta._permission`。请把 `cb` 改为设置 `_btn`，并把 `route.meta.permission` 的读取改成 `route.meta._permission`。
::
