`normalizePermissionMenus` 把后端原始菜单树转换成路由形状：

- 在 `cb` 里标注 `_btn: true` 的节点会**折叠**进父级的 `meta._permission`（key = `permission` 字段，value = 整个按钮节点，`_btn` 标记会被剥离）
- 有 menu 子节点的节点视为**分组**，`redirect` 到它的第一个子节点
- `cb` 返回 falsy 值会**丢弃**该节点及其子树
- 后端的 `meta` 对象会被**摊平**进 `meta`

```ts
const tree = normalizePermissionMenus(menus, v => ({ ...v, _btn: `${v.type}` === '2' }))
```

第二个参数 `cb` 是可选的；当后端数据已是正确形状、且没有按钮权限需要折叠时可省略。

::callout{icon="i-lucide-lightbulb"}
`cb` 会作用在**每个节点**上——它是调整**任意字段**的唯一入口:改名、增删、转换都在这里做。标 `_btn`、把 `name` 改到 `title`(见下)、把数字 `type` 归一——全都只是 `cb` 里的字段调整。
::

## 保留字段

库的所有字段都以 `_` 前缀，与你的后端数据隔离：

| 字段 | 位置 | 含义 |
| :-- | :-- | :-- |
| `_btn` | 输入（你在 `cb` 里设置） | 标记一个节点为待折叠的按钮 |
| `meta._permission` | 输出 | 折叠到父级上的按钮权限 |
| `meta._external` | 输出 | 标记为外链（渲染成 `<a>`，不注册路由） |

## 菜单 path 处理

- **外链**（`http(s)://`、协议相对的 `//`）保留在树里但不注册——请渲染成 `<a>`（它们带有 `meta._external`）。
- **参数页**：后端 path 支持中括号或冒号两种写法——`/detail/[id]` 与 `/detail/:id` 都能命中 `pages/detail/[id].vue`。
- **带 query**：`/report?range=7d` 注册 pathname `/report`，同时保留完整值供侧边栏链接使用。
- **去重**：同一页面被多个菜单项指向时只注册一次（首个生效）；菜单树里仍保留每个入口。

::callout{icon="i-lucide-info"}
**只有出现在后端菜单里的页面才会被注册。** 不作为菜单项的详情/编辑页——比如菜单里只有 `/menu`,却有 `pages/menu/[id].vue`——会保持被摘除、访问 404。要让它可达,把它的路由也放进后端菜单,通常作为**隐藏项**:在 `cb` 里打标(如 `hidden: true`),渲染侧边栏时跳过这类节点。用 `static` 白名单也行,但会让它变公开(无鉴权)。
::

## route 层字段与 `name` 覆盖

属于 vue-router 路由记录合法字段的那些——`path`、`name`、`redirect`、`alias`、`props`、`beforeEnter`、`sensitive`、`strict`（以及你通过 `routeFields` 选项额外加的）——会保留在 **route 层**;节点上的其余字段进 `meta`。

::callout{icon="i-lucide-triangle-alert" color="warning"}
`name` 就是其中之一,所以后端的 `name` 会**覆盖页面自己的路由 name**(取值为 `node.name ?? entry.name`——后端优先)。vue-router 的路由 name 必须**唯一**,而这里去重是按 **path**、不是 name。若两个菜单项 `name` 相同(比如同样的显示文案),vue-router 会报 duplicate name 警告,按 name 导航(`navigateTo({ name })`)也会歧义。
::

如果你后端的 `name` 是**显示文案**(而非唯一标识),在 `cb` 里把它挪走,别让它变成路由 name:

```ts
normalizePermissionMenus(menus, v => ({
    ...v,
    title: v.name, // 显示文案 → meta.title(非 route 层字段)
    name: undefined, // 不覆盖路由 name → 用页面自己的
    _btn: `${v.type}` === '2',
}))
```

侧边栏改用 `meta.title` 渲染。或者:后端 `name` 用唯一标识(如 `sys-user`),再单独给一个 `title` 做显示。

## 从 1.x 迁移到 2.0

::callout{icon="i-lucide-triangle-alert" color="warning"}
**2.0 破坏性改动:**
- `normalizeMenus` → **`normalizePermissionMenus`**(函数本身改名)
- `setMenusList` → **`setMenuList`**(`usePermissionState()` 与数据源上下文里)
- 按钮改为在 `cb` 里用 **`_btn`** 标注,不再从 `type === 'button'` 推断
- 折叠的权限从 `meta.permission` → **`meta._permission`**

即:把函数名 / setter 调用改掉,`cb` 里设 `_btn`,并读 `route.meta._permission`。
::
