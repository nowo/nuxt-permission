`hasPermission` 是自动导入的，可直接在模板里用。**没有** `v-permission` 指令——原生 `v-if` 本身是响应式的、可与 `v-else` 组合，无需额外机制。

```vue
<template>
    <button v-if="hasPermission('menu-add')">
        新增
    </button>
    <button v-if="hasPermission(['menu-edit', 'menu-view'])">
        编辑
    </button>        <!-- 任一命中 -->
    <button v-if="hasPermission(['menu-edit', 'menu-view'], true)">
        编辑
    </button>  <!-- 全部命中 -->
</template>
```

## 签名

```ts
function hasPermission(perm: PermissionKey | PermissionKey[], isAll?: boolean): boolean
```

- 单个 key，或 key 数组。
- `isAll`（默认 `false`）：任一命中即可。`true`：每个 key 都要命中。
- **空数组**始终返回 `false`——空的权限要求不授予任何访问。

## 读取状态

```ts
const { permissions } = usePermissionState()
// permissions 是 Ref<PermissionKey[]> —— 扩展 PermissionMap 后会收窄为你的联合类型
```

::callout{icon="i-lucide-shield-alert" color="warning"}
`hasPermission` 只是体验层。真正的鉴权请在后端强制执行。
::
