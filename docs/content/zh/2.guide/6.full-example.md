## 1. 配置

```ts [nuxt.config.ts]
export default defineNuxtConfig({
    modules: ['nuxt-permission'],
    permission: {
        // 只有这些保持静态;其余页面全部转为动态(登录后注册)
        static: ['/', '/login'],
    },
})
```

## 2. 后端数据形状

服务端是权威来源:返回当前用户的扁平 `permissions` 和 `menus` 树。这里 `type: 1` 是菜单,`type: 2` 是按钮。

```ts [server/api/user.get.ts]
export default defineEventHandler((event) => {
    const auth = getHeader(event, 'authorization')
    if (!auth?.startsWith('Bearer ')) {
        throw createError({ statusCode: 401, statusMessage: 'Unauthorized' })
    }

    return {
        permissions: ['menu-add', 'menu-edit', 'menu-view'],
        menus: [
            { id: 1, name: '仪表盘', path: '/dashboard', type: 1, children: [] },
            {
                id: 2,
                name: '菜单管理',
                path: '/menu',
                type: 1,
                meta: { sort: 1 },
                children: [
                    { id: 3, name: '新增', permission: 'menu-add', type: 2 },
                    { id: 4, name: '编辑', permission: 'menu-edit', type: 2 },
                    { id: 5, name: '删除', permission: 'menu-delete', type: 2 },
                ],
            },
        ],
    }
})
```

## 3. token 存取(由你负责)

库完全不碰登录态。一个小 composable 让 token 响应式并持久化:

```ts [composables/useToken.ts]
export function useToken() {
    const token = useState<string>('token', () => (import.meta.client ? localStorage.getItem('token') ?? '' : ''))

    function setToken(value: string) {
        token.value = value
        if (!import.meta.client) return
        value ? localStorage.setItem('token', value) : localStorage.removeItem('token')
    }

    return { token, setToken }
}
```

## 4. 数据源

`app/permission.ts` 读 token、拉后端、写状态、返回菜单树。库在首屏/刷新时调它;你在登录后用 `load()` 调它。

```ts [app/permission.ts]
export default definePermissionSource(async ({ setPermissionList, setMenuList }) => {
    const { token } = useToken()
    if (!token.value) return [] // 未登录 → 不注册任何路由

    const { permissions, menus } = await $fetch('/api/user', {
        headers: { authorization: `Bearer ${token.value}` },
    })

    setPermissionList(permissions)

    // 取消下一行注释,用一个「拥有全部权限」的账号登录,把浏览器控制台打印的这行,直接粘到 `PermissionMap`(第 8 步),即可获得完整的 key 补全:
    // console.log('keys: ' + permissions.map(p => `'${p}'`).join(' | '))

    // type 2 = 按钮 → 用 _btn 标注,使其折叠进父级 meta._permission
    const tree = normalizePermissionMenus(menus, v => ({ ...v, _btn: `${v.type}` === '2' }))
    setMenuList(tree)
    return tree
})
```

## 5. 登录与登出

```vue [pages/login.vue]
<script setup lang="ts">
const { setToken } = useToken()
const { load } = usePermissionState()

async function onLogin() {
    const { token } = await $fetch('/api/login', { method: 'POST' })
    setToken(token)
    await load() // 跑数据源、写状态、注册路由
    await navigateTo('/dashboard')
}
</script>

<template>
    <button @click="onLogin">登录</button>
</template>
```

```ts
// 登出,任意处
const { clear } = usePermissionState()
const { setToken } = useToken()

function onLogout() {
    setToken('')
    clear() // 清空状态 + 注销动态路由
    navigateTo('/login')
}
```

## 6. 布局:用 `menus` 渲染侧边栏

用响应式的 `menus` 状态渲染导航(它在路由注册之后才被赋值,所以链接始终可解析):

```vue [app.vue]
<script setup lang="ts">
const { menus } = usePermissionState()
const { token } = useToken()
const isLoggedIn = computed(() => !!token.value)
const navMenus = computed(() => menus.value.filter(m => m.path))
</script>

<template>
    <div>
        <nav>
            <NuxtLink to="/">首页</NuxtLink>
            <template v-if="isLoggedIn">
                <NuxtLink v-for="m in navMenus" :key="m.path" :to="m.path">{{ m.name }}</NuxtLink>
            </template>
        </nav>
        <NuxtPage />
    </div>
</template>
```

## 7. 用 `hasPermission` 守卫 UI

按钮级权限已折叠进 `meta._permission`;用 `v-if="hasPermission(...)"` 守卫:

```vue [pages/menu.vue]
<script setup lang="ts">
const route = useRoute()
</script>

<template>
    <div>
        <!-- 删除按钮被隐藏:'menu-delete' 不在已授予的权限里 -->
        <button v-if="hasPermission('menu-add')">
            {{ route.meta?._permission?.['menu-add']?.name }}
        </button>
        <button v-if="hasPermission('menu-edit')">编辑</button>
        <button v-if="hasPermission('menu-delete')">删除</button>
    </div>
</template>
```

## 8. 类型(可选但推荐)

扩展可合并接口,让 key 有补全、会收窄 —— 见[类型扩展](/zh/guide/type-extension):

```ts [types.d.ts]
declare module 'nuxt-permission' {
    interface PermissionMap {
        keys: 'menu-add' | 'menu-edit' | 'menu-view' | 'menu-delete'
    }
    interface PermissionButton {
        id: number
        name: string
        permission: string
    }
}

export {}
```
