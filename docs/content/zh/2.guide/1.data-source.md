创建 `<srcDir>/permission.ts`（文件名即 `source` 选项，默认 `permission`）。它 default export 一个数据源：读取登录态、拉取权限 / 菜单、写入状态，并返回菜单树。**库在首屏 / 刷新时调用它，你在登录后调用 `load()`——两次跑的是同一个数据源。**

```ts [app/permission.ts]
export default definePermissionSource(async ({ setPermissionList, setMenusList }) => {
    const token = /* 你自己的登录态 */ useCookie('token').value
    if (!token) return []

    const { permissions, menus } = await $fetch('/api/user', {
        headers: { authorization: `Bearer ${token}` },
    })

    setPermissionList(permissions)

    // 用保留标记 `_btn` 标注按钮，使其折叠进父级的 meta._permission
    const tree = normalizeMenus(menus, v => ({ ...v, _btn: `${v.type}` === '2' }))

    setMenusList(tree)
    return tree
})
```

`definePermissionSource`、`normalizeMenus` 以及各 composable 都是自动导入的。

## 登录后注册

token 存取由你负责；登录成功后调用 `load()`，再导航：

```ts
const { load } = usePermissionState()

async function onLogin() {
    await $fetch('/api/login', { method: 'POST' }) // 你：拿到并存好 token
    await load() // 库：运行数据源、写入状态、注册路由
    await navigateTo('/dashboard')
}
```

::callout{icon="i-lucide-info"}
`load()` 仅限客户端。SSR 首屏走 `router.options` 注册，所以你永远不需要在 SSR 期间调用 `load()`。
::

## 登出

```ts
const { clear } = usePermissionState()

function onLogout() {
    clear() // 清空权限 / 菜单状态，并注销动态路由
}
```
