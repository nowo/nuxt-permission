## 类型

从 `'nuxt-permission'` 导入。大多通过声明合并扩展 —— 见[类型扩展](/zh/guide/type-extension)。

| 类型 | 说明 |
| :-- | :-- |
| `PermissionMap` | 空接口;**扩展它**来声明你的权限 key |
| `PermissionKey` | 扩展后为 `PermissionMap` 值的联合,否则回退为 `string` |
| `PermissionButton` | 折叠后按钮节点的形状(`meta._permission` 的 value);扩展以定型其字段 |
| `PermissionMenu` | `normalizeMenus` 产出的菜单节点(route 层字段 + `children` + `meta`) |
| `PermissionSource` | 数据源签名:`(ctx) => PermissionMenu[]`(同步或异步) |
| `PermissionSourceContext` | 注入数据源的 `ctx`:`{ setPermissionList, setMenusList }` |

## 模块配置项

在 `nuxt.config.ts` 的 `permission` 键下配置 —— 完整表格见[配置](/zh/getting-started/configuration)。

| 选项 | 类型 | 默认 |
| :-- | :-- | :-- |
| `enabled` | `boolean` | `true` |
| `static` | `string \| string[]` | `['/', '/login']` |
| `source` | `string` | `'permission'` |
| `routeFields` | `string[]` | `[]` |
| `group` | `'redirect' \| 'navigate'` | `'redirect'` |
