`runtime/composables` 下的一切都是**自动导入**的——模板和脚本里无需 import 即可使用。

## `definePermissionSource`

```ts
function definePermissionSource(source: PermissionSource): PermissionSource
```

数据源(`<srcDir>/permission.ts` 的 default export)的类型助手。原样返回传入的 source,仅用于给 `ctx` 参数定型。见[数据源](/zh/guide/data-source)。

## `hasPermission`

```ts
function hasPermission(perm: PermissionKey | PermissionKey[], isAll?: boolean): boolean
```

判断当前用户权限,响应式。

- `perm` —— 单个 key 或 key 数组。
- `isAll` —— `false`(默认):任一命中即可;`true`:每个 key 都要命中。
- **空数组**始终返回 `false`。

见[权限控制](/zh/guide/permission-control)。

## `normalizeMenus`

```ts
function normalizeMenus<T = any>(
  list: T[],
  cb?: (item: T) => any | undefined | null | false
): PermissionMenu[]
```

把后端原始菜单树转换成路由形状。在 `cb` 里标注 `_btn: true` 把按钮折叠进父级 `meta._permission`;返回 falsy 值丢弃该节点及其子树。`cb` 可选。见 [normalizeMenus](/zh/guide/normalize-menus)。

## `usePermissionState`

```ts
function usePermissionState(): {
  permissions: Ref<PermissionKey[]>
  menus: Ref<PermissionMenu[]>
  routesVersion: Ref<number>
  hasPermission: (perm: PermissionKey | PermissionKey[], isAll?: boolean) => boolean
  setPermissionList: (list: string[]) => void
  setMenusList: (tree: PermissionMenu[]) => void
  load: () => Promise<void>
  clear: () => void
}
```

权限 / 菜单状态与其操作。

| 成员 | 类型 | 说明 |
| :-- | :-- | :-- |
| `permissions` | `Ref<PermissionKey[]>` | 当前权限列表(响应式;扩展 `PermissionMap` 后收窄) |
| `menus` | `Ref<PermissionMenu[]>` | 当前菜单树(用于渲染侧边栏) |
| `routesVersion` | `Ref<number>` | 每次注册路由后自增;在 `computed` 里依赖它来重读 `useRouter().getRoutes()`(后者非响应式) |
| `hasPermission` | `(perm, isAll?) => boolean` | 与独立函数同一实现 |
| `setPermissionList` | `(list: string[]) => void` | 写权限状态(在数据源里调用) |
| `setMenusList` | `(tree: PermissionMenu[]) => void` | 写菜单状态 |
| `load` | `() => Promise<void>` | 重跑数据源、写状态、注册路由(登录后 / 手动刷新)。**仅客户端** |
| `clear` | `() => void` | 清空状态并注销 `load()` 注册的动态路由(用于登出) |

### 读取最新的完整路由列表

`useRouter().getRoutes()` 返回的是非响应式快照。动态注册后要重读它,需依赖 `routesVersion`:

```ts
const { routesVersion } = usePermissionState()
const routeList = computed(() => {
    void routesVersion.value      // 依赖:addRoute 之后自增 → 重算
    return useRouter().getRoutes() // 此刻是最新的
})
```
