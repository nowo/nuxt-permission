在 `nuxt.config.ts` 的 `permission` 键下配置模块：

```ts [nuxt.config.ts]
export default defineNuxtConfig({
    modules: ['nuxt-permission'],
    permission: {
        enabled: true,
        static: ['/', '/login'], // 公开静态路由白名单；其余全部转为动态
        // source: 'permission',  // 数据源文件（相对 srcDir，省扩展名）
        // routeFields: [],        // 额外保留在 route 层的字段
        // group: 'redirect',      // 分组节点自带页面时：'redirect'（默认）| 'navigate'
    },
})
```

## 选项

| 选项 | 类型 | 默认 | 说明 |
| :-- | :-- | :-- | :-- |
| `enabled` | `boolean` | `true` | 总开关；为 `false` 时所有页面保持静态、消费方调用不会抛错 |
| `static` | `string \| string[]` | `['/', '/login']` | 白名单；**其余全部转为动态**。glob：`/home/**` 公开整个子树 |
| `source` | `string` | `'permission'` | 数据源文件（相对 srcDir，省扩展名） |
| `routeFields` | `string[]` | `[]` | 在内置 `RouteRecordRaw` 字段之外，额外保留在 route 层的字段 |
| `group` | `'redirect' \| 'navigate'` | `'redirect'` | 分组节点（有 menu 子节点的菜单）且自带页面时：`redirect` 到第一个子节点，或 `navigate` 渲染自身页面。无自带页面的分组一律重定向 |

## 白名单

`static` 作为 glob 与 `route.path` 匹配。所有**未**命中的页面都会变成动态路由，只有后端菜单注册后才存在。

```ts
static: [
    '/', // 精确匹配
    '/login', // 精确匹配
    '/public/**', // 整个 /public 子树，含 /public 本身
]
```
