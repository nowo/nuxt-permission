**nuxt-permission** 为 Nuxt 带来后端菜单 / 权限驱动的动态路由：非白名单页面在构建期从静态路由表中摘除，登录后再从后端菜单注册——在此之前这些路由根本不存在。它内置权限状态、`hasPermission` 和菜单转换助手，类型可通过声明合并扩展。

## 工作原理

1. 构建期，所有**不在** `static` 白名单里的页面会从路由表中移除，并收集进一个代码分割的 `import()` manifest。
2. 首屏（刷新 / 直达链接）时，库在 `router.options` 里运行你的数据源——SSR 安全，在首次导航前完成。
3. 登录后你调用 `load()`，它运行同一个数据源、写入权限状态，并从后端菜单注册路由。

## 库负责的部分

- 构建期摘除非白名单路由、运行时注册
- 权限 / 菜单状态（`usePermissionState`）
- 菜单转换助手（`normalizePermissionMenus`）
- 体验层的权限判断 `hasPermission`

## 由你负责的部分

- token 存取、登录 / 登出端点
- 未登录用户的跳转（库只保证非白名单路由无法到达）

::callout{icon="i-lucide-shield-alert" color="warning"}
`hasPermission` 只是**体验层，不是安全边界**——真正的鉴权始终在后端。
::
